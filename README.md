# Vipoo - AI 驱动的播客创作平台

Vipoo 是一个 AI 播客创作平台，让播客从「录音 + 剪辑」，变成「像文档一样可编辑的创作流程」。

## 🚀 快速开始

### 安装依赖

```bash
npm install
```

### 启动开发服务器

```bash
npm run dev
```

打开 [http://localhost:3000](http://localhost:3000) 查看首页。

## 📁 项目结构

```
Vipoo/
├── app/                    # Next.js App Router
│   ├── layout.tsx         # 根布局
│   ├── page.tsx           # 首页
│   └── globals.css        # 全局样式
├── components/            # React 组件
│   ├── Hero.tsx          # 首页 Hero 区域
│   └── FeatureBlocks.tsx # 功能展示区块
└── package.json          # 项目配置
```

## ✨ 功能特性

### 首页功能
- ✅ Hero 区域：主标题、副标题、CTA 按钮
- ✅ 视觉展示：AI 播客相关插画/抽象图形
- ✅ 三个核心功能块：
  - AI 驱动播客生成
  - 基于文本的音频编辑
  - AI 智能增强与修复

### 创作入口页 (`/create`)
- ✅ 路径选择：主题/提示词生成 vs 基于内容生成
- ✅ 主题生成表单：输入主题/提示词，选择音色
- ✅ 内容生成表单：输入文本或链接，选择音色
- ✅ 音色选择：单人/双人、音色风格（专业/轻松/温暖/活力）

### 播客编辑页 (`/editor/[id]`) - 核心功能
- ✅ 文本编辑器（左侧）：
  - 段落级文本显示和编辑
  - 点击段落跳转到对应音频位置
  - 删除段落功能
  - 拆分段落功能
  - 合并段落功能
- ✅ 音频播放器（右侧）：
  - 音频播放控制
  - 进度条显示
  - 段落时间轴
  - 文本-音频同步高亮
- ✅ 段落级音频生成：
  - 每个段落独立音频文件
  - 段落ID与音频文件绑定
  - 修改文本后标记需要重新生成
  - 支持单独重新生成段落音频

### AI 音质增强模块 (`/api/podcast/enhance`)
- ✅ 去噪功能：移除背景噪音和杂音
- ✅ 音量平衡：自动调整音量，确保各段落音量一致
- ✅ 清晰度增强：提升语音清晰度和可懂度
- ✅ 增强对话框：可视化选择增强选项
- ✅ 进度显示：实时显示处理进度

### 导出模块 (`/api/podcast/export`)
- ✅ 格式选择：支持 MP3 和 WAV 格式
- ✅ 质量选择：低/中/高三种质量选项
- ✅ 增强选项：可选择使用增强后的音频
- ✅ 导出对话框：完整的导出配置界面
- ✅ 下载功能：导出完成后直接下载

## 🛠️ 技术栈

- **Next.js 14** - React 框架（App Router）
- **TypeScript** - 类型安全
- **Tailwind CSS** - 样式框架
- **React** - UI 库
